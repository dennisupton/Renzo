<Renzo>

A terminal-based HTML editor that makes building websites easy

python main.py index.html
Features 01
🌲 Node Tree View

See your entire HTML structure as a live collapsible tree, right in the terminal.

🔍 Property Inspector

Select any tag and edit its attributes inline. Supports strings and boolean props.

🏷️ Tag Search

Browse all HTML tags by name with descriptions and insert them instantly.

🌐 Live Browser Preview

Your page hot-reloads at 127.0.0.1:5090 as you edit. No refresh needed.

💾 Save to File or Clipboard

Ctrl+S writes back to your file if one was passed, otherwise copies to clipboard.

⌨️ Fully Keyboard Driven

No mouse needed. Navigate, edit, move, and restructure nodes with keys alone.

Install 02

Requires Python 3.8+. Install dependencies with pip:

pip install blessed flask pyperclip

Then run on any HTML file:

python main.py index.html

Or run without an argument to start with a blank template (saves to clipboard):

python main.py
Note: You also need a htmlTags.json file in the project root — a list of {"tag": "...", "description": "..."} objects used by the tag search panel.

The terminal splits into two panels side by side:

Left panel — the node tree. Navigate and edit text nodes inline.

Right panel — the property inspector for the selected tag.

Workflow Example 05

Adding a styled button to your page from scratch:

  1. Navigate to the body node in the tree
  2. Press n to open tag search, type button, press Enter to insert it
  3. Press Enter on the new button node to open the property inspector
  4. Press n to add a class property, type btn, press Enter
  5. Press Esc to return to the tree
  6. Press i to insert inner text, type Click me, press Enter
  7. Check your browser at 127.0.0.1:5090 to see the result live
  8. Press Ctrl+S to save
Project Structure 06
renzo/ ├── main.py # entry point, render loop, keyboard input ├── inspector.py # node tree, property editor, HTML parser, node classes ├── file.py # HTML state, serialisation, file/clipboard save ├── network.py # Flask server + SSE stream for live browser preview ├── tagSearch.py # tag search panel ├── htmlTags.json # tag database used by search └── renzo.png # project logo
How It Works 07

Renzo parses your HTML into a flat list of node and inner objects, each carrying an indent level representing nesting depth. When you make edits, file.convertToString() rebuilds the full HTML string from that list. network.py then streams the update to the browser over SSE so the preview refreshes automatically without any page reload.

Known Limitations 08
File loading: Pass a file path as an argument (python main.py index.html) to load your own HTML. Without an argument, Renzo starts with a blank template and saves to clipboard instead of disk.
Large documents: Very large HTML files may cause rendering slowness due to full-screen terminal redraws on every frame.
Group move edge cases: Ctrl+Shift+Up/Down has some known bugs with deeply nested structures.
